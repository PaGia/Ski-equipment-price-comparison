# 開發工作流程

## 工具分工說明

**Claude Code for VS Code**
**主要職責**: 程式碼建構與實作
- ✅ 撰寫新功能程式碼
- ✅ 修改現有程式碼
- ✅ 重構與優化
- ✅ Bug 修復實作
- ✅ 檔案建立與編輯
- ✅ 程式邏輯實現
- ✅ **實作完成後執行 git commit**
- 🗑️ **實作完成後刪除測試用檔案**
- 🤔 **工作原則**: 慢慢想 (think hard)、不清楚先問

### GitHub Copilot
**主要職責**: 規劃與除錯分析

#### 規劃階段 (Planning)
- ✅ 需求分析與討論
- ✅ 技術方案評估
- ✅ 架構設計建議
- ✅ 開發計畫制定
- ✅ 技術選型建議
- ✅ **制定解決規劃與實作步驟**
- ✅ **評估技術可行性與資源需求**
- ✅ **設計系統改進方案**
- ✅ **規劃測試策略與驗證方法**
- ❌ **不負責**: 實際程式碼實作
- 🤔 **工作原則**: 慢慢想 (think hard)、不清楚先問

#### 除錯階段 (Debug)
- ✅ 錯誤日誌分析
- ✅ 問題根因調查
- ✅ 系統狀態檢查
- ✅ 完整除錯報告
- ✅ 解決方案建議
- ✅ **深入程式碼邏輯分析**
- ✅ **識別設計缺陷與邏輯漏洞**
- ✅ **追蹤問題影響範圍**
- ✅ **評估修復複雜度與風險**
- ❌ **不負責**: 實際修復實作
- 🤔 **工作原則**: 慢慢想 (think hard)、不清楚先問

## 標準工作流程

### 1. 需求提出
```
用戶提出需求 → GitHub Copilot 分析 → 產出規劃文件
```

### 2. 規劃階段
```
GitHub Copilot:
├── 需求分析
├── 技術評估
├── 架構設計
├── 實作計畫
└── 風險評估
```

### 3. 實作階段
```
Claude Code for VS Code:
├── 程式碼撰寫
├── 功能實現
├── 測試驗證
├── 文件更新
├── 刪除測試用檔案
└── Git commit (實作完成後)
```

### 4. 除錯階段
```
問題發現 → GitHub Copilot 診斷 → Claude 修復實作
```

## 協作原則

### 🎯 通用工作原則
- **慢慢想 (Think Hard)**: 每個步驟都要深思熟慮
- **不清楚先問**: 遇到不確定的情況務必先詢問澄清
- **品質優於速度**: 確保理解需求後再執行

### 📁 **報告儲存規範**
- **所有 Planning 和 Debug 報告儲存於**: `development-reports/` 資料夾
- **Planning 報告**: `development-reports/planning/` 
- **Debug 報告**: `development-reports/debug/`
- **檔案命名**: 遵循 `YYYY-MM-DD_描述_類型.md` 格式

### 📋 規劃階段
- GitHub Copilot 負責需求分析與計畫制定
- 產出詳細的實作指引給 Claude Code
- 不直接進行程式碼編寫
- **執行前**: 仔細思考需求，不清楚時主動詢問
- **報告輸出**: 儲存於 `development-reports/planning/`

### 💻 實作階段  
- Claude Code for VS Code 根據計畫進行實作
- 專注於程式碼品質與功能實現
- 遵循既定的架構設計
- **實作完成後刪除測試用檔案**
- **實作完成後必須執行 git commit**
- **執行前**: 深入理解需求與計畫，疑問時先詢問澄清

### 🐛 除錯階段
- GitHub Copilot 進行問題診斷與分析
- 產出完整的除錯報告
- Claude Code 根據報告進行修復
- **診斷前**: 仔細分析問題現象，不明確時先問清楚
- **報告輸出**: 
  - 除錯分析報告儲存於 `development-reports/debug/`
  - 解決方案規劃儲存於 `development-reports/debug/`

## 溝通格式

### 規劃文件格式
```markdown
# 功能規劃: [功能名稱]

## 需求分析
- 目標與期望
- 使用情境
- 成功指標

## 技術方案
- 架構設計
- 技術選型
- 實作步驟

## 風險評估
- 潛在問題
- 解決策略
```

### 除錯報告格式
```markdown
# 除錯報告: [問題描述]

## 問題現象
- 錯誤訊息
- 復現步驟
- 影響範圍

## 根因分析
- 問題定位
- 相關程式碼
- 邏輯分析

## 解決建議
- 修復方案
- 預防措施
- 測試建議
```
